RH Assistant Frontend Implementation Plan
1. Project Initialization & Configuration
Tooling: Initialize using Create React App (no Vite).
Styling: Install and configure Tailwind CSS for rapid UI development and pixel-perfect design matching.
Dependencies:
react-router-dom (v6) for navigation.
lucide-react for clean, modern SVG icons matching the design.
recharts for the circular pipeline charts.
axios for API communication with the backend.
clsx and tailwind-merge for dynamic class management.
Design Tokens: Configure tailwind.config.js to include the specific color palette from the mockups:
Primary Indigo (#6366f1)
Background Gray (#f9fafb)
Success Green (#22c55e)
Warning Orange (#f59e0b)
Danger Red (#ef4444)
2. Directory Architecture
text
src/
├── assets/          # Images, logos, global CSS
├── components/      # Reusable UI building blocks
│   ├── layout/      # Sidebar, TopBar, MainLayout
│   ├── ui/          # Buttons, Cards, Badges, Modals
│   └── icons/       # Custom SVGs if needed
├── pages/           # Main route components
│   ├── Dashboard/   # Home page with widgets
│   ├── Mails/       # Smart Inbox split view
│   ├── Candidates/  # Candidate list and detailed profile
│   └── JobPostings/ # Job management
├── services/        # API call wrappers (axios)
├── utils/           # Helper functions (formatting dates, text)
└── App.js           # Router configuration
3. Core Layout Implementation (The Skeleton)
MainLayout Component: Create a wrapper that contains the Sidebar on the left and a main <main> area on the right.
Sidebar: Fixed width, vertical navigation links with active state styling (light purple background with bold text). Includes the "Upgrade to Pro" banner and User Profile snippet at the bottom.
TopBar: Flexible width, contains the global search input, notification bell, and current page title dynamically populated based on the route.
4. Page 1: Home Dashboard (Overview)
Header Section: Welcome message ("Good morning, Sarah!").
Quick Summary Widgets: Two wide cards for "Reminders" and "My Mails (Smart HR Inbox)".
Metrics Grid: Four summary cards displaying Total Candidates, New Applications, Interviews Scheduled, and Offers Accepted, including percentage trend indicators.
Charts & Lists:
Top Candidates: A table-like list showing avatar, name, position, and a visual progress bar for the Match Score.
Recruitment Pipeline: Use Recharts to build the central donut chart showing the breakdown of candidates by stage (Applied, Screening, Interview, etc.).
5. Page 2: My Mails (Smart Inbox)
Split Layout: CSS Grid or Flexbox to divide the screen 40/60.
Left Pane (Mail List):
Filter/Sort header.
Scrollable list of email cards showing sender, subject, date, and AI Priority badges (High, Medium, Low).
Right Pane (Mail Reader):
Header with Quick Actions (Mark Unread, Star, Delete).
Email body rendering.
AI Extracted Information Card: A visually distinct section presenting the parsed JSON data (Name, Phone, Skills array, Category, Match Score).
Action Buttons: "Add to Candidates", "Reject", "Reply".
6. Page 3: Candidate Profile (Detail View)
Header Card: Large component featuring candidate photo, name, applied role, contact info, and a prominent circular Match Score graph.
Tab Navigation: Switch between Overview, CV & Documents, Timeline, and Notes.
Overview Tab Sections:
AI Summary: Text block displaying the parsed summary.
Skills: Responsive grid of tags/badges.
Experience & Education: Timeline-style list.
Skill Gap Analysis: Side-by-side comparison of matching skills vs missing skills.
Right Sidebar (Status & Actions):
Application details (Source, Date).
Recruitment Status Stepper (Applied -> Screening -> Interview -> Offered).
CV Document preview/download card.
Quick Actions grid.
7. Integration & State Management
Create a simple React Context or use standard prop drilling for global states like user authentication or sidebar collapse.
Connect the services/api.js to our Mongoose/Express backend:
GET /api/dashboard/stats for the Home page.
GET /api/emails for the Mails page.
GET /api/candidates/:id for the Candidate profile.
8. Final Polish
Ensure fully responsive design for tablets and smaller screens.
Add hover states, subtle transitions, and loading skeletons for a premium feel.
